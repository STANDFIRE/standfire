FUEL VALIDATION UN-VALIDATION

These actions are available only if the user is the owner of the fuel.

VALIDATION

To be used in a FireTec export, a fuel have to be validated in the database. It means that all compulsary data have been saved and that verifications rules are respected.

On top left, a panel display the fuel general informations.

On top right, a panel display the fuel shape general informations. This shape will be validated with the fuel. For a plant this shape should be 3D.

The white panel display the database check rapport informations.

4 cases:

  • the rapport return value is OK : validation procedure is POSSIBLE.
  • the rapport return value is WARNING : some data does not respect verification rules BUT validation procedure is POSSIBLE.
  • the rapport return value is ERRORS : some data does not respect verification rules AND validation procedure is IMPOSSIBLE
  • verification is IMPOSSIBLE or there is no shape stored for this fuel. Validation procedure is IMPOSSIBLE

Available actions are :

  • Save in database: to validate the fuel if it is POSSIBLE
  • Cancel: to close the window

UNVALIDATION

To be modified, a fuel have to be un-validated first.

On top left, a panel display the fuel general informations.

Available actions are :

  • Save in database: to unvalidate the fuel
  • Cancel: to close the window
