FUEL DESACTIVATION OR REACTIVATION

This interface aims at plant and layer desactivation or reactivation in the database.

These actions are available only if the user is the owner of the fuel.

FUEL DESACTIVATION

The fuel informations appears on the screen for verification.

After validation, this fuel and all attached shapes (sample, 2D or 3D shapes) will be desactivated, it means that utilisation of this fuel will be denied in virtual scenes creation or in exportation for models as FireTec.

Data wonâ€™t be physically deleted, but only logically deactivated.

  • Save in database : to validate the desactivation.
  • Cancel : to cancel and close the window
FUEL REACTIVATION

The fuel informations appears on the screen for verification.

After validation, the fuel will be available again in the database.

Only the plant or the layer will be reactivated, not the shapes !

  • Save in database : to validate the reactivation.
  • Cancel : to cancel and close the window
