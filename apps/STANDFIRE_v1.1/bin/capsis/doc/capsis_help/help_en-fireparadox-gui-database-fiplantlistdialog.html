FUEL LIST

All fuel objects (plants or layers) stored in the database appear in this list.

The result table contains for each fuel : the team, the site, the species name, the crown dimensions , the fuel origin (measured or vistual) and the mention if the plant is validated and desactivated or not.

The list is sorted by team - site - species - height - diameter.

Selectors on top of the screen can reduice the list size.

Control buttons are:

  • Add: to add a new fuel
  • Shapes: to manage shapes attached to this fuel
  • Parameters: to modify this fuel parameters
  • Validate/Unvalidate: to validate or unvalidate the fuel
  • Edit: to edit an existing fuel
  • Copy: to copy this fuel into a new one
  • Desactivate/Reactivate: to desactivate or reactivate an existing fuel
  • Close: to close the window

Edit action is usefull to display fuel informations. Modification will be possible if the user is the owner of the fuel and if this fuel is neither desactivated or validated. In the contrary, the fuel will have be be unvalidated or reactivated before being updated again.

Validate action is usefull to check a fuel data integrity in the database and to permit utilisation in virtual scene creation and in exportation for models as FireTec. To be validated a plant must have a 3D shape and a layer must have a 2D shape. A validated fuel cannot be modified.

Unvalidate action is usefull to cancel a fuel validation.

Desactivate action is usefull to denied utilisation of a fuel without deleting referenced data. Desactivation is also done for all shapes attached to the fuel.

Reactivate action is usefull to calcel a desactivation. Only the plant or the layer will be reactivated, but not the shapes.
