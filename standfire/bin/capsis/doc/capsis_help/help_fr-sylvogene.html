2. Objectifs

Module pour l'aide au suivi permanent de la ressource forestière nationale

Utiliser des données de l'Inventaire Forestier National en entrée (ancienne ou nouvelle méthode) ; choisir le module de croissance le plus adapté pour simuler la dynamique de croissance des arbres et du peuplement recensé ; choisir la durée ou la période de simulation ; générer des fichiers en sortie utilisables dans les bases de données de l'IFN.

3. Description

Sylvogène est un programme spécial à l’intérieur de Capsis ; il n’a pas exactement la structure habituelle des modules de Capsis. Ce programme ressemble plus à un script qui demande l’intervention d’un utilisateur à son démarrage. Son rôle est de lancer des simulations avec un ou plusieurs modules de Capsis.

On ouvre la boîte de dialogue de cette interface Capsis - Sylvogène à partir d’une fenêtre de commande MS-DOS dans laquelle on exécute la commande suivante :

C:\capsis4\capsis.bat -p script sylvogene.SylvoStarter

Cette fenêtre de dialogue permet :

a) d’aller rechercher les deux fichiers de données d’entrée (un fichier au niveau peuplement et un fichier au niveau arbre) nécessaires à la simulation et générés par l’IFN.

Les fichiers de données sont des fichiers de texte au format CSV. Un fichier CSV est un fichier texte (par opposition aux formats dit « binaires »). Chaque ligne correspond à une ligne du tableau et les cellules d'une même ligne sont séparées par une virgule. Les fichiers comportent une ligne avec les en-têtes. Dans le fichier «placette», chaque ligne qualifie une placette et dans le fichier «arbre», chaque ligne qualifie un arbre. Un document IFN-INRA interne a été élaboré et décrit l’ensemble des variables, leurs définitions, leur unité et leur précision.

b) de vérifier le nombre d’enregistrements valides dans le fichier décrivant les peuplements IFN sélectionnés et dans le fichier décrivant les arbres de ces peuplements.

c) de sélectionner les modèles de croissance disponibles pour réaliser des simulations

d) de sélectionner et de hiérarchiser les modèles de croissance implémentés dans Capsis selon leur priorité lors d’une simulation

e) de choisir une durée de simulation ou une date finale de simulation.

f) de choisir une méthode de simulation (deux méthodes de simulation existent).

Méthode 1

Cette méthode de simulation du massif permet une simulation de la croissance de chaque peuplement individuellement. Dans chacun des modules peuvent être proposées un ou plusieurs scénarios d’éclaircies et méthodes pour réaliser ces éclaircies. Ce mode est utile pour déboguer le comportement d’un modèle face à une placette particulière, mais également pour traiter des scénarios prospectifs à attribuer à certaines placettes selon leurs caractéristiques.

Méthode 2

Cette méthode de simulation du massif permet de prendre en compte les caractéristiques générales du massif pour effectuer les coupes rases et les éclaircies. Ce mode est utile dans les études de prospective ou des scénarios de récolte et de boisement peuvent être testés.

g) de calculer le nombre de peuplements IFN compatibles selon l’ordre de sélection des modèles

h) de fournir l’état d’avancement des simulations ainsi qu’un bilan final.

Trois types de fichiers sont enregistrés en fin de simulation dans le répertoire temporaire de l’utilisateur, par exemple sous Windows : C:\Documents and Settings\nomUtilisateur\Local Settings\Temp

Chaque peuplement traité pendant la simulation est répertorié dans 3 fichiers :

• NomDuModule_Numéro.sylvogene : fichier initial comportant la ligne décrivant le peuplement et éventuellement les lignes décrivant les arbres.

• NomDuModule_ Numéro.sylvogene.csv : fichier texte donnant les résultats des variables peuplements et arbres après le dernier pas d’évolution.

• NomDuModule_ Numéro.sylvogene.prj : fichier projet de Capsis permettant d’ouvrir les simulations sous le mode interactif de Capsis.

D’autre part, quatre fichiers de résultats sont enregistrés dans le répertoire temporaire relatif au répertoire d'installation de capsis (***\capsis4\tmp) :

sylvogene_stand.csv sylvogene_tree.csv

Ces deux fichiers permettent de concaténer les résultats des simulations soit au niveau peuplement soit au niveau arbre. Dans le cas du mode 1, seul le résultat final de la simulation est enregistré (donc 1 peuplement – 1 ligne). Dans le cas du mode 2, les résultats sont enregistrés chaque année et pour la durée de simulation choisie.

sylvogene_scrap_stand.csv sylvogene_scrap_tree.csv

Ces deux fichiers contiennent les peuplements et leurs arbres rejetés par le ou les modules choisis. Cela signifie que leurs caractéristiques empêchent leurs traitements par un des modules de croissance en raison de leur localisation, de la composition en espèces ou de leurs caractéristiques dendrométriques. Ces points sont donc traités par l’IFN.

4. Validité

Testé sur les donnés IFN des 4 régions forestières nationales qui constituent le massif landais avec les modules PP3 et Lemoine.
