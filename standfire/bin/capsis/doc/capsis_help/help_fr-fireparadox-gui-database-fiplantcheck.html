VALIDATION DEVALIDATION D'UN COMBUSTIBLE

Ces actions ne sont possibles que sur les combustibles dont l'utilisateur est le propriétaire.

VALIDATION

Pour être placé sur une scène virtuelle ou utilisé dans un export FireTec, un combustible doit être validé dans la base de données, c'est à dire que l'on va vérifier que toutes les données obligatoires ont été saisies et qu'elles respectent les règles de validité qui ont été fixées.

En haut de l'écran à gauche, un cadre rappelle les informations générales concernant le combustible.

En haut de l'écran à droite, un cadre rappelle les informations générales concernant la forme qui va être validée avec la plante ou la nappe. Pour une plante il s'agit obligatoirement d'une forme 3D.

Le cadre blanc affiche le résultat de la vérification effectuée dans la base de données.

4 possibilités :

  • Le rapport renvoi la valeur OK : la validation est POSSIBLE.
  • Le rapport renvoi un ou plusieurs WARNING : certaines données ne respectent pas les règles de validité, mais la validation de la plante est néanmoins POSSIBLE.
  • Le rapport renvoi une ou plusieurs ERREURS : certaines données ne respectent pas les règles de validité, la validation de la plante est IMPOSSIBLE.
  • La vérification est IMPOSSIBLE ou aucun forme n'existe pour la plante : la validation de la plante est IMPOSSIBLE.

Les actions relatives au boutons sont:

  • Enregistrer dans la base: pour valider le combustible si c'est POSSIBLE
  • Fermer: pour fermer la fenêtre

DEVALIDATION

Pour être modifié, un combustible doit pas être préalablement dévalidé.

En haut de l'écran à gauche, un cadre rappelle les informations générales concernant le combustible.

Les actions relatives au boutons sont:

  • Enregistrer dans la base: pour dévalider le combustible
  • Fermer: pour fermer la fenêtre
