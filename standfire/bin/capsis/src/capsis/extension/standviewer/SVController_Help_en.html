Capsis Hep

Principle

This viewer is compatible with the models managing a list of trees (individuals or cohorts). It shows a table containing a line per tree data structure found in the scene. For each tree, the following columns are proposed: tree identifier, age, diameter, height, number, marqued, x, y, z, species. Each column is filled when possible: if the trees of the model do not contain some corresponding property, the column remains empty.

Features

The total number of trees is cumulated in the field "N". The number of tree objects is shown in the field "Lines". The two values are equal if the model manages individuals, they can be different in the case of cohorts where a tree object can represent several individuals, in this case check the "Number" column.

The tree list can be sorted in ascending or descending order on the proposed properties by clicking (resp. ctrl-clicking) on the corresponding column header.

It is possible to restrict the tree list to a group of trees by changing the "group" component. The list is auto-updated at each change.

It is also possible to select some trees in the list to check them more accurately by choosing a selection mode in the proposed list and by triggering the "Show selection" button. When a tree is selected in the list, a short description also appears in the description field at the bottom of the viewer.
